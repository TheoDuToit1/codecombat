## 🧠 Game Curriculum Structure


---

## Coding Course: 160 Levels

### **Course Structure**
- The course is divided into **160 sequential levels**.
- Each level is a self-contained lesson, designed to teach or reinforce a specific coding concept or game mechanic.
- Levels are defined in files like `src/data/lessonPlan.ts`, `src/data/levels.ts`, and `src/data/gauntletLevels.ts`.

### **What’s in a Level?**
Each level typically includes:
- **Starter Code**: Pre-filled code that gives the player a starting point.
- **Objectives**: Clear goals (e.g., “Collect the key and reach the exit”).
- **Hints**: Optional tips to help the player if they get stuck.
- **Story/Context**: Narrative elements to make the lesson engaging.
- **Map/Grid**: A unique dungeon layout with obstacles, enemies, collectibles, and an exit.
- **Allowed Commands**: Sometimes, only certain coding commands are available to encourage learning specific concepts.

### **Progression**
- Players must use code to solve each level’s challenge (e.g., move the hero, attack, collect items).
- When the objectives are met, the level is marked as complete, and the next lesson is unlocked.
- Progress is tracked, and players can see how far they’ve come (e.g., “Lesson 42/160”).

### **Learning Path**
- Early levels focus on basic movement and simple commands.
- As the player advances, levels introduce:
  - Loops and conditionals
  - Functions and parameters
  - More complex enemy AI and interactions
  - Resource management (health, food, keys)
  - Advanced game mechanics (generators, power-ups, etc.)
- The course is designed to build coding skills step by step, with increasing difficulty and complexity.

---

## Gauntlet Mode

### **What is Gauntlet Mode?**
- **Gauntlet** is an advanced, arcade-style mode inspired by classic Gauntlet games.
- It’s unlocked after completing a certain number of lessons (e.g., after lesson 40).
- Gauntlet mode is less about step-by-step instruction and more about applying everything the player has learned in a fast-paced, challenging environment.

### **How Gauntlet Mode Works**
- The player is dropped into a large, procedurally generated or hand-crafted dungeon.
- The goal is to survive as long as possible, collect treasures, defeat enemies, and reach the exit.
- **Enemy generators** spawn new enemies over time, increasing the challenge.
- The player’s health and food decrease over time, so they must find resources to survive.
- The player can use all the coding skills learned in the course to automate their hero’s actions, create strategies, and optimize their code for survival and high scores.

### **Key Features**
- **Endless or Multi-Level**: Gauntlet mode can be endless or have a series of increasingly difficult levels.
- **Score Tracking**: Players earn points for defeating enemies, collecting treasures, and surviving longer.
- **Replayability**: Each run is different, encouraging players to refine their code and strategies.
- **Bosses and Special Events**: Advanced levels may include boss fights or unique challenges.
